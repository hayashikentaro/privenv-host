# Examples

These examples are copyable starting points for local development. Every value is intentionally fake and must be replaced by the Host operator when a real project needs local secrets.

Do not use realistic credentials in examples, tests, docs, manifests, responses, or audit logs.

## privenv.host.json

`privenv.host.json` is Host-owned configuration. Guests should not read it directly.

```json
{
  "version": "0.1",
  "capabilities": [
    {
      "id": "cmd.npm.test",
      "kind": "command",
      "description": "Run npm test.",
      "command": {
        "program": "npm",
        "args": ["test"]
      },
      "env": [],
      "timeoutMs": 30000,
      "redactionPolicy": "default"
    },
    {
      "id": "cmd.example.with.env",
      "kind": "command",
      "description": "Run npm test with an example env reference.",
      "command": {
        "program": "npm",
        "args": ["test"]
      },
      "env": [
        {
          "name": "EXAMPLE_SERVICE_TOKEN",
          "source": "secret",
          "exposedToGuest": false
        }
      ],
      "timeoutMs": 30000,
      "redactionPolicy": "default"
    }
  ]
}
```

The config may name env vars, but it must not contain raw env values.

## privenv.vault.example.json

`privenv.vault.example.json` is a root-level example file. It is safe to commit only because the values are obviously fake placeholders.

```json
{
  "version": "0.1",
  "secrets": {
    "EXAMPLE_DATABASE_URL": {
      "value": "example-secret-placeholder-not-real",
      "classification": "secret"
    },
    "EXAMPLE_USER_EMAIL": {
      "value": "example-pii-placeholder-not-real",
      "classification": "pii"
    },
    "EXAMPLE_SERVICE_TOKEN": {
      "value": "example-token-placeholder-not-real",
      "classification": "token"
    }
  }
}
```

## .privenv/vault.json

`.privenv/vault.json` is Host-owned and gitignored. Guests must never read it directly.

This example still uses fake placeholders. A real local file would be created manually by the Host operator from `privenv.vault.example.json`.

```json
{
  "version": "0.1",
  "secrets": {
    "EXAMPLE_SERVICE_TOKEN": {
      "value": "local-example-token-placeholder-not-real",
      "classification": "token"
    }
  }
}
```

## privenv.manifest.json

`privenv.manifest.json` is Guest-readable and generated by the Host.

```json
{
  "version": "0.1",
  "capabilities": [
    {
      "id": "cmd.npm.test",
      "kind": "command",
      "description": "Run npm test.",
      "command": {
        "program": "npm",
        "args": ["test"]
      },
      "env": []
    },
    {
      "id": "cmd.example.with.env",
      "kind": "command",
      "description": "Run npm test with an example env reference.",
      "command": {
        "program": "npm",
        "args": ["test"]
      },
      "env": [
        {
          "name": "EXAMPLE_SERVICE_TOKEN",
          "source": "secret",
          "exposedToGuest": false
        }
      ]
    }
  ]
}
```

The manifest can include env names, but it must never include env values or vault values.

## EffectRequest

The Guest sends an `EffectRequest` with a capability ID only. It does not send command strings, env values, or secret requests.

```json
{
  "id": "req_001",
  "type": "effect.request",
  "capabilityId": "cmd.npm.test",
  "metadata": {
    "guestName": "example-guest",
    "guestRunId": "example-run-001",
    "reason": "example local development request"
  }
}
```

## EffectResponse

The Host returns a redacted `EffectResponse`.

```json
{
  "requestId": "req_001",
  "type": "effect.response",
  "ok": true,
  "result": {
    "exitCode": 0,
    "stdout": "fake execution complete\n",
    "stderr": "",
    "redactions": []
  },
  "auditId": "audit_req_001"
}
```

If a fixture secret appears in fake execution output during tests, the response should contain `[REDACTED]` instead of the raw fixture value.

## Validation Result

`privenv-host validate` emits machine-readable JSON.

```json
{
  "ok": true,
  "type": "validation.result",
  "config": {
    "exists": true,
    "valid": true
  },
  "vault": {
    "exists": false,
    "valid": true
  },
  "capabilities": [
    {
      "id": "cmd.npm.test",
      "valid": true,
      "envNames": [],
      "missingEnvNames": [],
      "errors": [],
      "warnings": []
    }
  ],
  "warnings": []
}
```

Missing vault entries are reported by env name only. Secret values are not printed.

## Doctor Output

`privenv-host doctor` emits human-readable setup status.

```text
Host config: OK
Vault: MISSING
Capabilities: 3 valid
Missing env: none
Real execution: not implemented
Mode: simulate only
```

Doctor output must not contain secret values.
